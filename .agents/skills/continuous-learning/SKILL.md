# Skill: Continuous Learning

Автоматизирует накопление полезных паттернов из каждой сессии.

## Когда использовать

В конце каждой сессии или после интересного решения — записать урок, чтобы следующая сессия не изобретала колесо заново.

## Структура записи

Все записи идут в `.codex/antigravity_memory.md` → секция `## Уроки и Паттерны`:

```markdown
### <YYYY-MM-DD>: <Краткое название>
- **Контекст:** Что делали
- **Проблема:** Что пошло не так или что было неочевидно
- **Решение:** Что помогло
- **Применять:** При <условии>
```

## Типовые паттерны, уже усвоенные в mudro

### 2026-03-17: WSL vs PowerShell для Docker
- **Контекст:** Запуск `make` и `go test` из Windows
- **Проблема:** `go test ./...` падает в PowerShell без Go в PATH
- **Решение:** Все Go/Make/Docker команды — через WSL; PowerShell — только для git, scp, файловых операций
- **Применять:** Всегда при Go или Docker командах

### 2026-03-17: SSH ключи Windows → VPS
- **Контекст:** SSH на VPS из PowerShell
- **Проблема:** WSL-ключи из `/mnt/c` дают ошибку прав 0777
- **Решение:** Использовать `ssh -i $env:USERPROFILE\.ssh\id_rsa admin@<ip>`
- **Применять:** При любых SSH/SCP операциях с VPS

### 2026-03-17: Vite CSS import без TypeScript декларации
- **Контекст:** Импорт `.css` файла в `App.tsx`
- **Проблема:** IDE показывает ошибку "Cannot find module"
- **Решение:** Это ложная ошибка IDE — Vite обрабатывает CSS нативно; `npm run build` проходит без ошибок
- **Применять:** Игнорировать CSS-импорт lint ошибки если `npm run build` успешен

### 2026-03-16: Безопасное удаление TG дублей
- **Контекст:** TG-посты дублировались из разных источников CSV+JSON
- **Проблема:** Прямое удаление могло сломать foreign keys
- **Решение:** `cmd/tgdedupe` — сначала переносит FK (comments, media, reactions), затем удаляет дубль
- **Применять:** Всегда при дедупликации постов

```markdown
- **Применять:** 
```

## Автоматическая проверка в конце сессии

Перед завершением сессии ответь себе:
1. Была ли неожиданная ошибка или поведение? → запиши урок
2. Использовался ли нестандартный обходной путь? → запиши паттерн
3. Было ли что-то существенно оптимизировано? → запиши в `done.md`
