# Улучшения по итогам логов (2026-03-16)

Источник: анализ `.codex/logs/20260316-1109/index.md` и событий `/find` из `.codex/tg_control.jsonl`.

## Приоритет P0 — надежность и безопасность

1. **Автопроверка экспонирования Postgres наружу**
   - Проблема в логах: БД была доступна на `0.0.0.0:5433`, в логах фиксировались внешние auth/SQL abuse попытки.
   - Улучшение: добавить preflight-проверку в `make health`/`make dbcheck`, которая валит прогон при bind не на loopback.
   - Метрика готовности: в CI и локально есть явный `PASS` для правила `db_port_is_loopback_only`.

2. **Запрет DSN с суперпользователем в runtime**
   - Проблема в логах: сервисы стартовали с DSN на `postgres`.
   - Улучшение: добавить проверку в загрузчик конфигурации (или startup guard), которая блокирует запуск API/ботов при `user=postgres` вне dev-режима.
   - Метрика готовности: сервисы не стартуют с небезопасным DSN, есть понятная ошибка в логах.

3. **Fail-fast в systemd на обязательных env-переменных**
   - Проблема в логах: `mudro-reporter.service` падал из-за отсутствующего `REPORT_CHAT_ID`.
   - Улучшение: добавить `ExecStartPre`-валидатор env (или встроенную валидацию в `cmd/reporter`/`cmd/bot`) с четким списком обязательных переменных.
   - Метрика готовности: при отсутствии env сервис завершает запуск с диагностикой до входа в crash-loop.

## Приоритет P1 — операционная устойчивость

4. **Идемпотентный деплой без stale-файлов**
   - Проблема в логах: `mudro-bot.service` падал из-за stale `cmd/bot/gototg.go`, которого нет в репозитории.
   - Улучшение: деплой через `git fetch && git reset --hard <commit>` (или чистый rsync с `--delete`) вместо частичных копирований.
   - Метрика готовности: zero stale `.go` после выката, bot/reporter поднимаются без ручной чистки.

5. **Единый runbook для host-vs-container DSN**
   - Проблема в логах: `docker compose exec ... psql "$DSN"` давал `connection refused`, потому что host DSN передавался в контейнер.
   - Улучшение: добавить явный раздел в runbook с матрицей команд (`host psql` vs `container psql`) и анти-примерами.
   - Метрика готовности: нет повторов этой ошибки в `.codex/state.md`/логах следующих сессий.

6. **Политика хранения исходных экспортов для сверки**
   - Проблема в логах: отсутствовал raw `vk_wall_*.json`, из-за чего не удалось сделать полную source-to-DB сверку.
   - Улучшение: завести обязательный каталог архивов вне репозитория + чеклист доступности сырья перед audit/import.
   - Метрика готовности: для каждого импорта есть путь к raw-данным и checksum в служебном логе.

## Приоритет P2 — качество кода

7. **Снижение числа пакетов без тестов**
   - Проблема в `/find`: фиксировались пакеты без тестов (10–12 по `go test ./...`).
   - Улучшение: ввести квартальную цель покрытия критичных пакетов (`internal/api`, импортеры, backfill-команды) и добавить smoke-тесты на ошибки конфигурации.
   - Метрика готовности: количество пакетов без тестов снижается релиз-к-релизу.

8. **Регулярный отчет по инцидентам из `journalctl`/docker logs**
   - Проблема в логах: ключевые проблемы обнаруживались уже после падений сервисов.
   - Улучшение: добавить `make ops-report` (последние N ошибок из systemd/docker + статус сервисов + сетевые бинды).
   - Метрика готовности: за 1 команду получаем сжатый health/security срез для дежурного.

4. Добавить в runbook раздел про DSN-контуры (host/container).
