## §1. `.github/workflows/ci.yml` оборван — CRITICAL

Файл содержит 62 строки и заканчивается так (hex-tail подтверждён):

```yaml
  backend-unit:
    name: Backend Unit
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Set up Go
        uses: actions/setup-go@v5
        with:
          go-version: '1.24'
      - name: Run backend unit tests
        run: go test -s
```

Что не так:
1. Команда `go test -s` бесполезна — `-s` не флаг `go test`. Либо шаг сфейлится, либо `-s` будет воспринят как имя пакета и получит "cannot find package".
2. Нет newline в конце файла.
3. Нет frontend job, хотя в памяти и вчерашнем отчёте CI описывался как "3 jobs, включая frontend (lint + build)". Проверка истории:
   ```
   f180ae3 2026-04-05 "fix: mvp hardening — rate limiting, casino fairness, frontend stability, proxy logging"
   ```
   С этого коммита файл в таком виде — либо неудачный rebase, либо прерванное редактирование. Вчерашний отчёт описывал CI *неточно*.
4. golangci-lint тоже выпал: есть job `backend-lint`, но он запускает `golangci/golangci-lint-action@v4` без чекаута go-модулей корректно — при этом `Set up Go` в этом job нет → lint скорее всего падает или использует встроенный Go из action. Смешано.

**Риск:** текущий `main` может проходить CI только потому, что job-ы "проходят" на кривых шагах (exit 0 по ошибке) или вся CI красная и это никто не замечает. Оба варианта опасны.

**Фикс (2-й критичный пункт этого дня):** переписать `ci.yml` на 3 рабочих джоба:
- `security` — gitleaks (как есть).
- `backend` — setup-go, `golangci-lint run`, `go test -short ./...`.
- `frontend` — setup-node, `cd frontend && npm ci && npm run lint && npm run test && npm run build`.

Автономно по Mission §3 (улучшение CI, не отключение проверок).

## §6. Sandbox limitation / index corruption

Попытка `git status` из sandbox:
```
error: bad signature 0x00000000
fatal: index file corrupt
```

Это повтор инцидента 2026-04-11 — Windows-mount не даёт sandbox безопасно работать с `.git/index`. Read-only команды (`git log`, `git branch -a`, `git diff` против коммитов) работают через pack-файлы и не трогают index. Write-команды из sandbox **запрещены** фидбэк-памятью.

**Владельцу:** перед пушем нужно восстановить index локально:
```powershell
cd E:\mudr\mudro11
del .git\index
git reset
git status
```
Это безопасно: index пересоздаётся из HEAD + working tree, untracked/modified файлы не теряются.
