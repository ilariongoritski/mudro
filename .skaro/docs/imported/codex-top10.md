# TOP10 изменений (ключевая ценность)

Правила:
- Хранить только действительно значимые изменения сути проекта.
- Список всегда до 10 пунктов, в порядке приоритета/влияния.
- При новом значимом изменении: добавить/поднять пункт, вытеснить наименее важный.

1. [feed-ui-v2] Лента `/feed` обновлена до полнофункционального просмотра: фильтры `Общая/VK/TG`, сортировка `Новые/Старые`, пагинация, медиа-блоки и ссылки на оригинал/обсуждение.
   Эффект: браузерная лента стала рабочим интерфейсом чтения контента по источникам.

2. [import-vk-full] Починен `cmd/vkimport` под текущую схему `posts` и загружен полный VK-архив.
   Эффект: в БД 1087 VK-постов с корректными `media`-полями в `jsonb`.

3. [import-tg-db] Добавлен `cmd/tgload` и реализован импорт TG `feed_items.json` в таблицы `posts/post_reactions`.
   Эффект: в БД 1094 TG-поста, фильтр `source=tg` в ленте показывает реальные записи.

4. [vercel-prod] Выполнен production deploy в Vercel; добавлен адаптер `pkg/vercelapi` для корректной сборки handler.
   Эффект: внешний URL активен и `/healthz` работает; для `/feed` осталось добавить production `DSN`.

5. [agent-mvp] Реализован MVP автономного агента: `migrations/002_agent_queue.sql` + `cmd/agent` + `internal/agent` + make-таргеты.
   Эффект: проект перешел от идеи к рабочему queue-контуру (planner/worker).

6. [ops-cloud-prod] Поднят прод-минимум в облаке: Postgres (docker compose) + mudro-api (systemd).
   Эффект: бэкенд работает 24/7 на VPS, `/healthz` стабильно возвращает `ok`.

7. [bot-security] Ограничение Telegram-команд по username `sirilarion`.
   Эффект: команды не исполняются посторонними, базовый контроль доступа закрыт.

8. [bot-health] Новый `/health` как дневная сводка (текущее состояние + проблемы/успехи + git-итоги).
   Эффект: оперативная диагностика проекта из телефона без ручного shell.

9. [runtime-events-limits] Добавлены runtime-контуры Kafka+Redis: публикация task-событий агента и distributed rate limiter API.
   Эффект: система готова к масштабированию по инстансам и устойчивой событийной обработке.

10. [agent-review-gate] Для очереди агента добавлен review-gate: статусы `waiting_approval/rejected`, approve/reject-команды и миграция `004_agent_review_gate.sql`.
   Эффект: risky-задачи теперь не исполняются worker-ом без явного подтверждения.
11. [frontend-fsd-foundation] Создан отдельный frontend-контур Mudro11 (`frontend/`) на React+TS+Redux Toolkit+RTK Query+Vite с FSD-архитектурой и hotpink-визуальной темой.
    Эффект: проект получил самостоятельный масштабируемый UI-слой вместо только серверного HTML-шаблона.
- 2026-03-06: Добавлен практический контур OpenClaw setup (scripts + runbook), позволяющий запустить инсталляцию на VPS без повторной ручной сборки шагов из статьи.
12. [openclaw-vps-base] На VPS поднят базовый контур OpenClaw + Mullvad VPN: контейнеры и конфиг инициализированы, egress подтвержден как US.
    Эффект: инфраструктурная основа для Clawbot готова; критический остаток — финальный стабильный запуск gateway.
13. [openclaw-telegram-stability] Выполнена живая диагностика дублей ошибок в Telegram: подтвержден рабочий VPN/gateway и внесен anti-overlap tuning (`timeout=3600`, `maxConcurrent=1`, `streaming=off`).
    Эффект: снижена вероятность повторных/накладывающихся ответов; `server_error` корректно классифицирован как внешний upstream-сбой.
14. [tg-root-vs-comments] На полном локальном TG-экспорте отделены root-посты автора `мудро` от reply-комментариев; JSON API и HTML-feed унифицированы по локальным media URL `/media/...`.
    Эффект: лента перестала превращать комментарии в посты, а TG-вложения начали стабильно отображаться на реальных данных.

- 2026-03-15 | Черновой MVP Mudro вынесен наружу: VPS API восстановлен, комментарий-media на сервере импортирован, а `frontend/` задеплоен на Vercel через proxy rewrites | Эффект: ленту уже можно открыть публично, не только локально.
- 2026-03-16 | [data-media-normalization] Введен аддитивный нормализованный media-слой (`media_assets`, `post_media_links`, `comment_media_links`) с backfill-командой и normalized-first API/import path.
  Эффект: проект получил нормальную эволюционную модель media без ломки текущего JSON API и без отдельной media-БД.
- 2026-03-16 | [test-safety-db] Устранен опасный integration test с implicit DSN: серверные/локальные тесты больше не трогают рабочую БД без явного isolated DSN.
  Эффект: снят один из самых рискованных источников случайной потери контентных данных в проекте.
- 2026-03-16 | [comment-model-normalization] Комментарийный слой доведен до нормальной модели: `parent_comment_id` + `comment_reactions`, а затем раскатан и подтвержден и локально, и на VPS.
  Эффект: у проекта теперь цельная эволюционная модель контента в обоих контурах — media и comment graph больше не держатся только на сыром JSON следе.
- 2026-03-16 | [vps-db-hardening] На VPS закрыт самый опасный инфраструктурный блокер: Postgres убран из открытого суперпользовательского режима в отдельный `mudro_app` + loopback bind + firewall guard на `tcp/5433`, а bot/reporter выровнены по актуальному дереву репы. Эффект: серверный контур API/ботов стал базово стабильным и перестал зависеть от публичного `postgres/postgres`.

- 2026-03-16 | [vps-ssh-hardening] На VPS окончательно закрыт root SSH по паролю и подтвержден безопасный базовый периметр (ail2ban, UFW, loopback-only Postgres, legacy WireGuard-unit снят из failed-state). Эффект: серверный контур Mudro перестал зависеть от password-login и стал заметно устойчивее к внешнему фону и случайным регрессам.
- 2026-03-16 | [openclaw-recovery] На VPS восстановлен рабочий OpenClaw-контур: обнаружен неверный state-dir mount (/root/.openclaw вместо /srv/openclaw-home/.openclaw), исправлены ownership и compose override, после чего gateway снова поднялся и Telegram-бот вернулся в живое состояние. Эффект: автономный серверный агент снова существует как рабочий runtime, а не как мертвый контейнер.

- 2026-03-16: фронтенд-черновик ленты переведен на живой локальный API :18080 и доведен до рецензируемого MVP-экрана с реальными метриками, thread-preview и drawer по TG-комментариям.
- 2026-03-16 | [public-mvp-reviewable] Публичный frontend на Vercel подтвержден как живой reviewable-MVP: health-check зеленый, страница отдает реальную ленту `1102 / 1 / 1101`, media и треды видны не только локально, но и на внешнем URL. Эффект: Mudro уже можно внешне рецензировать как хостящийся продукт, а не как локальную сборку.
- 2026-03-16 | [vps-data-sync] Публичный VPS-контур выровнен с локальным по данным и API: исправлен серверный `source=vk`, а CSV-импорты TG views/comments раскатаны прямо на сервер. Эффект: внешний сайт на Vercel теперь показывает актуальные VK-посты и точные просмотры/комментарии TG из серверной БД, а не только из локального контура.
- 2026-03-17 | [vps-frontend-self-hosted] Основной MVP frontend переведен на self-hosted контур на VPS: `nginx` раздает `frontend/dist` с `:80`, а `/api`, `/media`, `/healthz` проксирует на локальный `mudro-api`.
  Эффект: проект теперь можно открывать прямо с VPS по IP как цельный продукт, а Vercel перестал быть обязательным runtime-звеном.
