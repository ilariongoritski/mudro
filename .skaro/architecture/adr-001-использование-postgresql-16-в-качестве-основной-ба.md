# ADR-001: Использование PostgreSQL 16 в качестве основной базы данных

**Status:** proposed
**Date:** 2026-03-19

## Context
Проекту требовалась надежная, масштабируемая и многофункциональная реляционная база данных для хранения основных данных приложения (посты, комментарии, медиа, реакции), которые имеют сложные взаимосвязи и требуют строгой согласованности.

## Decision
PostgreSQL 16 был выбран за его надежность, расширенные функции (JSONB, полнотекстовый поиск, строгие гарантии транзакций), активное сообщество и проверенную историю использования в производственных средах. Он хорошо подходит для сложной реляционной модели данных проекта.

## Alternatives
1.  **MySQL**: Предлагает хорошую производительность и широкое распространение, но исторически имел менее полные расширенные функции (например, более мощную поддержку JSON, специфические типы индексации) и другие гарантии транзакций по сравнению с PostgreSQL. Отклонен из-за менее всеобъемлющего набора функций для сложных моделей данных и предпочтения ACID-совместимости и расширяемости PostgreSQL.
2.  **MongoDB (NoSQL)**: Предлагает гибкость схемы и горизонтальное масштабирование, но жертвует строгой реляционной целостностью и транзакционными гарантиями между несколькими документами, что критически важно для взаимосвязанного графа `posts`, `comments`, `media` и `reactions`. Отклонен, поскольку основная модель данных по своей природе реляционна и требует строгой согласованности.

## Consequences
- Positive: Высокая целостность данных, мощные возможности запросов, расширяемость, хорошая производительность, сильная поддержка сообщества, хорошо подходит для сложных реляционных данных.
- Negative: Требует тщательного проектирования схемы и управления миграциями; потенциально более высокие операционные издержки по сравнению с более простыми базами данных для очень базовых случаев использования.
- Risks: Возможность возникновения узких мест производительности при плохо оптимизированных запросах на больших наборах данных; требует специализированных знаний DBA для расширенной настройки.
