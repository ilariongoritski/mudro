# ADR-003: Использование React 19 + TypeScript 5 + Vite для разработки фронтенда

**Status:** proposed
**Date:** 2026-03-19

## Context
Проекту требовался современный, эффективный и поддерживаемый стек фронтенда для создания интерактивных пользовательских интерфейсов для контентной платформы.

## Decision
React 19 был выбран за его компонентную архитектуру, большое сообщество и проверенную историю. TypeScript 5 обеспечивает статическую типизацию для улучшения качества кода и удобства поддержки. Vite был выбран в качестве инструмента сборки за его быстрый сервер разработки и оптимизированный вывод сборки, улучшая опыт разработчика.

## Alternatives
1.  **Vue.js**: Предлагает аналогичный компонентный подход и хорошую производительность. Однако существующий опыт команды и более широкое отраслевое принятие/экосистема для React были более сильными факторами. Отклонен из-за меньшей существующей знакомости команды и немного меньшей экосистемы по сравнению с React.
2.  **Angular**: Комплексный фреймворк, предлагающий структурированный подход. Однако он часто сопряжен с более крутой кривой обучения и более жесткой структурой, чем React, что может быть излишним для первоначального объема проекта и предпочтительной гибкости. Отклонен из-за более высокой сложности и более жесткой структуры фреймворка.

## Consequences
- Positive: Быстрые циклы разработки, высокоинтерактивный пользовательский интерфейс, строгая типовая безопасность, большая экосистема компонентов, отличный опыт разработчика с Vite.
- Negative: Требует понимания жизненного цикла React и управления состоянием; потенциальная «усталость от JavaScript» из-за множества развивающихся библиотек.
- Risks: Быстрая эволюция экосистемы фронтенда может потребовать частых обновлений и рефакторинга; потенциальные проблемы с производительностью при неоптимизированном рендеринге компонентов.
