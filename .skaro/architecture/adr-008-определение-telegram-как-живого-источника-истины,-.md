# ADR-008: Определение Telegram как живого источника истины, VK как только снимка

**Status:** proposed
**Date:** 2026-03-19

## Context
Платформа интегрирует данные из нескольких внешних источников (Telegram, VK) и нуждается в четкой политике относительно того, какой источник активно поддерживается и считается каноническим для актуальности.

## Decision
Telegram был назначен живым источником истины для активной синхронизации и актуальности контента. Это означает, что усилия по разработке импортеров, обновлений и функций реального времени в основном сосредоточены на Telegram. VK поддерживается как источник только для снимков, что означает, что его данные импортируются и сохраняются, но не обновляются активно для актуальности, если только он явно не будет вновь открыт для активной синхронизации.

## Alternatives
1.  **Рассматривать Telegram и VK как живые источники**: Активно поддерживать и синхронизировать оба источника для актуальности. Отклонен из-за увеличения затрат на разработку и эксплуатацию, необходимых для управления двумя различными, активно меняющимися внешними API, что удваивает сложность для импортеров, дедупликации и обработки ошибок, что было нецелесообразно при текущих ресурсах.
2.  **Полностью отказаться от VK**: Удалить интеграцию и данные VK. Отклонен, поскольку данные VK по-прежнему имеют историческую ценность и вносят вклад в общую базу контента, даже если они не обновляются активно. Сохранение его только в виде снимка позволяет сохранять контент без постоянной нагрузки на обслуживание.

## Consequences
- Positive: Сфокусированные усилия по разработке на одном основном источнике, снижение сложности обслуживания внешних интеграций, четкая политика актуальности данных, эффективное распределение ресурсов.
- Negative: Контент VK со временем устареет; пользователи, полагающиеся на актуальность VK, не будут обслуживаться; потенциальное расхождение данных между источниками.
- Risks: Изменения в API Telegram могут нарушить основной поток данных; будущая необходимость повторной активации VK может потребовать значительной переработки; восприятие пользователями «устаревшего» контента VK.
