# output/

Эта папка предназначена только для локальных и одноразовых артефактов, которые не должны попадать в git.

Канонические подпапки:

- `output/db/` — one-shot планы, сверки, дампы проверки и JSON-артефакты по БД
- `output/deploy/` — временные deploy-проверки и артефакты rollout
- `output/doc/` — локально сгенерированные `.docx`/`.pdf` и черновые экспортные документы
- `output/import/` — временные копии CSV/JSON для импорта и сверки
- `output/logs/` — локальные логи dev/probe/proxy/vite/api
- `output/pdf/` — промежуточные PDF-артефакты
- `output/playwright/` — screenshots и browser smoke-артефакты
- `output/ssh-tool/` — локальные вспомогательные SSH-скрипты и node-зависимости для одноразовых ops-задач

Правила:

1. Не хранить здесь канонический исходный код.
2. Не ссылаться из кода на файлы внутри `output/` как на обязательные runtime-зависимости.
3. Если артефакт стал важной документацией, переносить его в `docs/` вручную в осмысленном виде.
4. Если артефакт нужен только на время сессии, складывать его в соответствующую подпапку `output/`, а не в корень репозитория.
