# TOP10 изменений (ключевая ценность)

Правила:
- Хранить только действительно значимые изменения сути проекта.
- Список всегда до 10 пунктов, в порядке приоритета/влияния.
- При новом значимом изменении: добавить/поднять пункт, вытеснить наименее важный.

1. [agent-mvp] Реализован MVP автономного агента: `migrations/002_agent_queue.sql` + `cmd/agent` + `internal/agent` + make-таргеты.
   Эффект: проект перешел от идеи к рабочему queue-контру (planner/worker) и готов к следующим итерациям gate/метрик.

2. [cloud-agent-vision] Зафиксирована целевая архитектура полностью автономного облачного агента (`planner+worker+gate+reporter`) для развития проекта.
   Эффект: есть практический план превращения репозитория в самоулучшающуюся систему на VPS.

3. [ops-cloud-prod] Поднят прод-минимум в облаке: Postgres (docker compose) + mudro-api (systemd).
   Эффект: бэкенд работает 24/7 на VPS, `/healthz` стабильно возвращает `ok`.

4. [bot-security] Ограничение Telegram-команд по username `sirilarion`.
   Эффект: команды не исполняются посторонними, базовый контроль доступа закрыт.

5. [bot-health] Новый `/health` как дневная сводка (текущее состояние + проблемы/успехи + git-итоги).
   Эффект: оперативная диагностика проекта из телефона без ручного shell.

6. [bot-llm] Команда `/mudro` с ответом LLM по контексту проекта (README/TODO/DONE).
   Эффект: быстрые проектные ответы и помощь по запуску/изменениям в одном чате.

7. [bot-followup] Поддержка выбора `1/2` после уточняющих ответов `/mudro`.
   Эффект: ветвление сценариев в Telegram без повторного набора длинных запросов.

8. [bot-actions] Переформатирование `/actions10` и `/actions1h` в осмысленные отчеты.
   Эффект: вместо сырых логов — структура «задача/проблемы/выполнения/доработки» и путь за час.

9. [bot-commits] Новый смысл `/commits3`: краткая суть 3 коммитов на русском.
   Эффект: быстрое понимание изменений без чтения полного `git log`.

10. [bot-memory] Постоянная память проекта: `.codex/todo.md` + `.codex/done.md` + команды `/todo`, `/todoadd`.
   Эффект: управляемый backlog и история выполненного прямо из Telegram.
