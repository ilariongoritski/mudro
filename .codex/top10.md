# TOP10 изменений (ключевая ценность)

Правила:
- Хранить только действительно значимые изменения сути проекта.
- Список всегда до 10 пунктов, в порядке приоритета/влияния.
- При новом значимом изменении: добавить/поднять пункт, вытеснить наименее важный.

1. [ops-cloud-prod] Поднят прод-минимум в облаке: Postgres (docker compose) + mudro-api (systemd).
   Эффект: бэкенд работает 24/7 на VPS, `/healthz` стабильно возвращает `ok`.

2. [bot-security] Ограничение Telegram-команд по username `sirilarion`.
   Эффект: команды не исполняются посторонними, базовый контроль доступа закрыт.

3. [bot-health] Новый `/health` как дневная сводка (текущее состояние + проблемы/успехи + git-итоги).
   Эффект: оперативная диагностика проекта из телефона без ручного shell.

4. [bot-llm] Команда `/mudro` с ответом LLM по контексту проекта (README/TODO/DONE).
   Эффект: быстрые проектные ответы и помощь по запуску/изменениям в одном чате.

5. [bot-followup] Поддержка выбора `1/2` после уточняющих ответов `/mudro`.
   Эффект: ветвление сценариев в Telegram без повторного набора длинных запросов.

6. [bot-actions] Переформатирование `/actions10` и `/actions1h` в осмысленные отчеты.
   Эффект: вместо сырых логов — структура «задача/проблемы/выполнения/доработки» и путь за час.

7. [bot-commits] Новый смысл `/commits3`: краткая суть 3 коммитов на русском.
   Эффект: быстрое понимание изменений без чтения полного `git log`.

8. [bot-memory] Постоянная память проекта: `.codex/todo.md` + `.codex/done.md` + команды `/todo`, `/todoadd`.
   Эффект: управляемый backlog и история выполненного прямо из Telegram.

9. [api-front] Единый фронтовый endpoint `GET /api/front` (meta + feed в одном ответе).
   Эффект: фронт получает стартовые данные одним запросом, проще интеграция.

10. [bot-feed] Команда `/feed5` для вывода 5 постов из `GET /api/front`.
    Эффект: проверка контента/API прямо из Telegram без открытия фронта.
