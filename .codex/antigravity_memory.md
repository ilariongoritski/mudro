## Уроки и Паттерны

### 2026-04-03: Сидирование MVP данных для UI
- **Контекст:** Разработка premium UI (dark glassmorphism) для MVP feed.
- **Проблема:** Отсутствие визуального контекста и контента сильно тормозит настройку стилей и проверку функционала ленты.
- **Решение:** Подготовка реалистичных mock/seed скриптов (рекламные посты + треды комментариев) до полировки UI.
- **Применять:** При разработке любых сложных UI-компонентов начинать с сидирования реалистичных данных.

### 2026-04-01: Разделение стейта в интерактивных UI (Emoji Casino)
- **Контекст:** Создание Telegram-style emoji слотов.
- **Проблема:** Рассинхрон между мгновенной генерацией результата на сервере (RTP профили) и длительной анимацией на фронтенде.
- **Решение:** Четкое разделение: бэкенд отдает конечные символы и обновляет persistent history API, фронтенд полностью изолирует логику вращения и останавливается на заданном сервером результате.
- **Применять:** При реализации любых интерактивных/игровых виджетов с серверным стейтом.

### 2026-03-25: Миграция сложного бэкенда в Go
- **Контекст:** Рефакторинг монолитного `server.go` в модульную архитектуру `internal/posts`, `internal/auth`.
- **Проблема:** Разрастание логики в едином файле (`server.go`) блокирует юнит-тестирование пайплайнов (например, media delivery) и усложняет роутинг.
- **Решение:** Модульное разделение по фичам (posts, auth) с самого начала, внедрение упрощенных механизмов (username-based) на ранних этапах для разблокировки UI/UX.
- **Применять:** При добавлении новых крупных фич в бэкенд сразу выделять их в пакет внутри `internal/`.
